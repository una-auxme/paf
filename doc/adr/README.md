# Architecture Decision Records (ADRs)

Use ADRs to preserve technical intent and trade-offs over time.

## Naming

- File format: `NNNN-short-title.md`
- Example: `0001-standardize-pytest-markers.md`

## Process

1. Copy `0000-template.md`.
2. Fill in context, options, decision, and consequences.
3. Link the ADR in the corresponding PR.
4. Mark superseded ADRs when decisions change.
